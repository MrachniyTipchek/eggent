## Cursor Cloud specific instructions

### Overview

Eggent is a local-first AI workspace (Next.js 15 + React 19 + TypeScript). All state is stored on disk in `./data/` — no external database. A single Next.js service runs on port 3000.

### Commands

Standard dev commands are defined in `package.json`:

- **Dev server:** `npm run dev` (port 3000)
- **Build:** `npm run build`
- **Lint:** `npx next lint` (the `npm run lint` script uses bare `eslint` which requires a flat config; use `npx next lint` for the legacy `.eslintrc.json` config)
- **Health check:** `curl http://localhost:3000/api/health`

### Auth

Default credentials are `admin` / `admin`. The app will prompt to change them on first login. Auth is cookie-based (HMAC-signed tokens via `EGGENT_AUTH_SECRET` in `.env`).

### Environment

The `.env` file is created from `.env.example`. Three secrets are auto-generated by the install scripts: `EXTERNAL_API_TOKEN`, `TELEGRAM_WEBHOOK_SECRET`, `EGGENT_AUTH_SECRET`. At least one LLM API key (`OPENAI_API_KEY`, `ANTHROPIC_API_KEY`, `GOOGLE_API_KEY`, or `OPENROUTER_API_KEY`) is required for AI agent features to work, but the app starts and serves the UI without one.

### Gotchas

- The `package.json` name is `design-vibe`, not `eggent`.
- Both `package-lock.json` and `pnpm-lock.yaml` exist; the canonical package manager is **npm** (the lockfile used by all install scripts is `package-lock.json`).
- `npm run build` uses `--turbopack --no-lint`; lint is not part of the build.
- Pre-existing lint warnings exist in vendor files under `src/lib/vendor/` — these are not regressions.
- The `data/` directory is git-ignored and created at runtime. Do not check it in.
